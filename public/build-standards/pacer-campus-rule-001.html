Campus Build Standard

Infrastructure
is declared once.

Behavior is inherited everywhere.

This rule governs how shared systems are structured across the PACER campus. It applies to theme engines, authentication, navigation, shared state, and design tokens. It applies to every room, every building, and every artifact that is part of this institution.

The Three Layers

Campus, Buildings, Artifacts

Every component of the campus belongs to exactly one layer. Each layer has a defined relationship to shared infrastructure.

Layer Owns Rule
Campus Infrastructure pacer-theme.js, authentication, navigation, shared state, design tokens Declares reality. Does not inherit.
Buildings (Rooms) Atrium, Archivist Hall, Business Center, Theater, future colleges Inherits from campus. Does not reinvent.
Standalone Artifacts Doctrine PDFs, exported reports, printable manifestos, offline documents Carries inline copies of required infrastructure. Expected to survive disconnected from campus.
Applied to Theme

One Theme Engine.
Two Deployment Modes.

Campus buildings:
  • Load <script src="/pacer-theme.js"></script> from campus root
  • Add data-pacer-toggle attribute to any toggle button
  • Do not define local theme variables or a competing toggle function
Standalone artifacts:
  • Carry an inline copy of PACERTheme in a <script> block
  • May be opened without any campus infrastructure present
  • Do not link to external pacer-theme.js
The Raccoon Warning
A room that defines its own theme engine has admitted a raccoon to the server closet. The damage is slow, incremental, and entirely preventable. When discovered, remove the competing engine immediately. Do not negotiate with the raccoon.
Never do this:
  • Load both inline and external pacer-theme.js on the same page
  • Define a RoomTheme, LocalTheme, or PageTheme in any building
  • Create a "special dark mode" for one room
  • Merge the inline and external versions into one "universal" file
Why This Rule Exists

The Pattern Runs
Through Everything

The Blue Pineapple is not recreated in every room. The institution defines hospitality. The rooms inherit it.

Doctrine is not rewritten for every building. The institution defines reality. The rooms inherit it.

The theme engine is not rewritten for every room. The institution defines appearance. The rooms inherit it.

Same pattern. Different layer.

Six months from now, a future builder will be tempted to define a special dark mode for one room because it seemed easier in the moment. This document exists to name that temptation clearly and explain why it leads to six months of nonsense afterward.

Infrastructure is declared once. Behavior is inherited everywhere. The institution teaches this. The rooms practice it.
